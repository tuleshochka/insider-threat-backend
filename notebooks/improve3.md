## 1. Самая серьезная проблема: нормализация `tsle` сделана с утечкой данных

Сейчас у тебя:

```python
X_tsle_raw = data.get('X_tsle', np.zeros_like(X, dtype=np.float32))
X_tsle_log = np.log1p(X_tsle_raw)
tsle_mean = X_tsle_log.mean()
tsle_std = X_tsle_log.std() + 1e-8
X_tsle = (X_tsle_log - tsle_mean) / tsle_std
```

### Почему это плохо
Ты считаешь `mean` и `std` **по всему датасету**, включая будущий test.

Это **data leakage**.  
Не самый катастрофический, как leakage таргета, но все равно методологически это ошибка.

### Как надо
1. Сначала сформировать окна
2. Потом разбить train/test по users
3. Потом **считать mean/std только на train**
4. И этими же параметрами нормализовать train и test

### Почему это важно
На защите или рецензии это могут заметить, и будет неприятно.

---

## 2. Валидация идет на `test_loader` прямо в цикле обучения

У тебя сейчас тест используется как validation:

```python
for epoch in range(NUM_EPOCHS):
    ...
    model.eval()
    ...
    for ... in test_loader:
        ...
```

### Это методологически неправильно
То, что ты называешь `test`, фактически используется для:
- выбора лучшей эпохи
- early stopping
- scheduler
- выбора лучшей модели

То есть это уже **validation**, а не test.

### Как надо
Разделить данные на:
- **train**
- **val**
- **test**

Причем все — **по пользователям**, а не случайно по окнам.

Например:
- 70% пользователей — train
- 15% — val
- 15% — test

И тогда:
- train — обучение
- val — early stopping, scheduler, threshold tuning
- test — только финальная оценка один раз

Сейчас это один из главных недостатков работы.

---

## 3. XAI-блок пока формально есть, но технически объяснение не совсем корректное

Сейчас ты делаешь:

```python
for layer in self.transformer.layers:
    seq_repr = layer(seq_repr)

attn_output, attn_weights = self.transformer.layers[-1].self_attn(
    seq_repr, seq_repr, seq_repr, average_attn_weights=True
)
trans_out = seq_repr
```

### В чем проблема
Ты извлекаешь attention **после того, как уже прогнала данные через все слои**, а потом еще раз вызываешь `self_attn` отдельно на уже трансформированном представлении.

Это не совсем те attention weights, которые реально использовались в исходном forward-pass при финальном решении.

То есть для иллюстрации — окей,  
но как **строгое XAI-объяснение** это пока спорно.

### Лучше сделать так
Либо:
- переопределить `TransformerEncoderLayer`, чтобы сохранять реальные attention weights во время forward;
либо
- использовать кастомный encoder layer с `need_weights=True`

Тогда можно честно сказать:
> “Извлечены реальные матрицы внимания последнего Transformer-слоя.”

Сейчас лучше в тексте не писать слишком категорично, а аккуратнее:
> “Для интерпретации использовалась приближенная визуализация междневных зависимостей на основе self-attention последнего слоя.”

---

## 4. Нет маскирования PAD на дневном уровне

У тебя дневная последовательность длиной `MAX_SEQ_LEN`, но LSTM получает паддинги как реальные элементы:

```python
_, (h_n, _) = self.day_lstm(lstm_in)
```

### Почему это проблема
LSTM проходит по всем 200 токенам, включая хвост из PAD.  
Да, embedding для PAD есть, но:
- `hour`
- `working_hours`
- `tsle`
тоже подаются для этих позиций

Их значения для PAD сейчас не маскируются явно как “невалидные”.

Это шум.

### Лучше
Использовать длины последовательностей:
```python
lengths = (x != 0).sum(dim=1).cpu()
packed = pack_padded_sequence(lstm_in, lengths, batch_first=True, enforce_sorted=False)
_, (h_n, _) = self.day_lstm(packed)
```

Это заметно улучшит корректность day-level encoder.
