## ⚠️ Критические проблемы и баги, которые ещё остались

### 🔴 1. БАГ в Sliding Window — утечка данных (data leakage)

```python
for i in range(len(indices) - WINDOW_SIZE + 1):
    w_idx = indices[i:i+WINDOW_SIZE]
    w_labels = labels[i:i+WINDOW_SIZE]
    ...
    win_y.append(1 if w_labels.sum() > 0 else 0)
```

**Проблема:** Если день D атакующий, то метку "1" получают **все 7 окон**, которые этот день содержат (окна, заканчивающиеся на D, D+1, ... D+6). Это раздувает число положительных примеров в 7 раз и **создаёт лёгкую задачу для модели** (а в реальности — нет).

**Что делать:**
- Либо помечать только окна, где атака происходит **в последний день окна** (`w_labels[-1] == 1`) — тогда это становится задачей предсказания "будет ли атака сегодня".
- Либо использовать **non-overlapping windows** (`stride=WINDOW_SIZE`).
- Либо вводить step > 1 (например, `stride=3`).

### 🔴 2. Padding для пользователей с <7 днями

```python
w_idx = np.pad(indices, (pad_len, 0), mode='edge')
```

Вы дублируете первый день, что **создаёт искусственные паттерны**. Лучше:
- Просто пропускать таких пользователей (`if len(indices) < WINDOW_SIZE: continue`).
- Или использовать `<PAD>` индекс с маскированием в Transformer.

### 🔴 3. Padding в `pos_encoder` инициализируется нулями

```python
self.pos_encoder = nn.Parameter(torch.zeros(1, window_size, hidden_dim))
```

Если оставить нули, обучаемый positional encoding начинается с нуля и теряет первоначальный смысл. Используйте:
```python
self.pos_encoder = nn.Parameter(torch.randn(1, window_size, hidden_dim) * 0.02)
```

### 🟡 4. Метрики обучения считаются по порогу 0.5

```python
preds_binary = (np.array(all_preds) > 0.5).astype(int)
```

С Focal Loss + сильным дисбалансом классов **порог 0.5 почти никогда не оптимален**. Это значит, что вы выбираете "лучшую модель" по неоптимальной метрике. Лучше:
- Считать **AUC-PR** (Average Precision) — он не зависит от порога.
- Сохранять модель по `best_pr_auc`, а не `best_f1@0.5`.

### 🟡 5. `model.eval()` запускается внутри цикла обучения, но статистика BatchNorm/Dropout не сбрасывается между fold-ами

Не критично, но если будете делать кросс-валидацию — нужно учесть.

### 🟡 6. Логирование атрибутов Attention/Transformer не используется

Модель возвращает `trans_out`, но он нигде не анализируется. **Огромный потенциал для XAI** — а это критично для темы "внутренняя безопасность".

## 🔧 Конкретные мини-правки в коде

### Исправление #1 (важное) — labeling в sliding window
```python
# Было:
win_y.append(1 if w_labels.sum() > 0 else 0)

# Стало (предсказание на последний день окна):
win_y.append(int(w_labels[-1]))
```
Это сделает задачу более реалистичной (предсказывать в реальном времени).

### Исправление #2 — early stopping
Сейчас 10 эпох без early stopping. Добавьте:
```python
patience_counter = 0
PATIENCE = 3
if f1 > best_f1:
    best_f1 = f1
    patience_counter = 0
else:
    patience_counter += 1
    if patience_counter >= PATIENCE:
        print("Early stopping!")
        break
```

### Исправление #3 — gradient clipping
В старой версии было, а в новой убрали. Для Transformer особенно важно:
```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

### Исправление #4 — нормализация tsle
Вы логарифмируете, но не нормализуете. Лучше:
```python
X_tsle = np.log1p(data.get('X_tsle', np.zeros_like(X)))
X_tsle = (X_tsle - X_tsle.mean()) / (X_tsle.std() + 1e-8)
```

### Исправление #5 — паддинг в day-level LSTM
LSTM не знает про PAD-токены. Добавьте маску или используйте `pack_padded_sequence`.
